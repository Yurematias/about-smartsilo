Monitoramento de sinais com JavaScript e Arduino

Neste tutorial, você verá como construir uma página web para monitoramento de sinais analóǵicos do Arduino, usando tecnologias da web, como JavaScript, HTML e CSS.

Para isso será utilizado o NodeJS para execução do JavaScript fora do navegador, o framework Johnny-five que possibilita fazer a comunicação serial entre o Node e a placa Arduino, e o Socket.io, biblioteca que será usada para enviar em tempo real os dados do Arduino para o gráfico no dashboard.

Tecnologias utilizadas

NodeJS

O NodeJS é basicamente um ambiente de execução JavaScript do lado servidor, permitindo a interpretação da linguagem fora do ambiente dos browsers. Sua criação abriu a possibilidade para desenvolver vários sistemas diferentes em JavaScript que vão desde programas desktop, servidores web complexos, até sistemas baseados em hardware como é o caso da aplicação a ser desenvolvida neste tutorial. NodeJS site oficial

Johnny-Five

Johnny-five é um framework que possibilita interagir e controlar hardware utilizando a linguagem JavaScript no NodeJS. Tendo seu lançamento em 2012 a biblioteca e a coleção de recursos são mantidas até hoje por uma comunidade ampla de desenvolvedores aficionados em robótica, hardware e software. Johnny-Five site oficial

Socket.io

O Socket.io é uma biblioteca JavaScript para aplicações web realizarem comunicação bidirecional, em tempo real e baseada em eventos entre clientes e servidores da web. Ele é constituído de duas partes, uma biblioteca do lado do front-end, a que é executada nos navegadores, e a do lado back-end que é executada no lado do servidor no NodeJS. A biblioteca dispõe de várias funcionalidades como detecção de desconexão e suporte à reconexão automática. Socket.io site oficial

Preparando o ambiente:

  • Instale a IDE do Arduino
  • Instale o Node.js
  • Instale o Visual Studio Express 2010 32 bit

  • Instale o Python 2.7.3
  • Abra o cmd e digite

  • Instale o node-gyp
    npm install -g node-gyp

Instalando pacotes no NodeJS

  • O Primeiro passo do tutorial é a criação da pasta do projeto. Então crie uma pasta para o projeto com um nome de sua preferência
  • Uma vez que já preparamos o ambiente para o Johnny-five funcionar precisamos instalá-lo na pasta do projeto, assim entre na pasta do projeto e execute o comando de terminal:
    npm install --save johnny-five
  • Também será necessário instalar o Express para criar nosso servidor web para isso digite no terminal:
    npm install --save express
  • Para instalação do Socket.io no NodeJS insira o comando:
    npm install --save socket.io

Preparando o Arduino

Antes de começarmos a codificar precisamos deixar o arduino no ponto para receber os sinais de voltagem dos potenciômetros, para isso siga o esquema a seguir colocando a saída dos potenciômetros nas entradas analógicas A0 e A1 do Arduino

Criando o servidor da aplicação

Criando JavaScript do back-end

Após toda essa configuração, finalmente podemos criar o primeiro arquivo, que será o back-end da aplicação. para deixar mais organizado podemos criar uma pasta denominada backend e assim criar nosso arquivo server.js dentro dela, portanto isso é opcional. assim crie o arquivo server.js e importe as devidas bibliotecas e frameworks anteriormente instaladas na pasta do NodeJS. Além disso vamos importar o módulo path do NodeJS que servirá para usarmos caminhos relativos do nosso sistema de arquivos 

                
const express = require("express"); // importando o Express 
const app = express();
const http = require("http").createServer(app);
const io = require("socket.io")(http); // importando Socket.io
const jFive = require("johnny-five"); // importando o Johnny-five
const path = require('path'); // será utilizado para fazer o express reconhecer o caminho

Logo após isso definimos uma constante referente a porta que o servidor será vinculado e definimos a pasta que será enviada nas requisições para o cliente neste caso criaremos uma pasta denominada frontend, onde estará JavaScript client-side, o CSS e o HTML. para fazer o servidor usar essa pasta como raiz de nossa página web insira no arquivo server.js o código 

                
// porta do servidor 
const port = 8000;
// atender requisições com pasta a frontend
app.use(express.static(path.resolve(__dirname + "/../frontend")));

Criando JavaScript do front-end

Obter front-end JavaScript completo

Uma vez criado o HTML, podemos criar o script que será vinculado a essa página, onde terá toda a configuração referente aos gráficos, além da manipulação da DOM

Primeiramente criaremos as variaveis a serem utilizadas 

                
const socket = io(); // constante que armazenará o objeto do socket.io
let value = 0; // valores de y inseridos nos gráficos
let pause = false; // determina se o gráfico está pausado
let x = 0; // x representa os pontos no eixo x do gráfico
let executingGraph; // armazenará dos dois gráficos

Antes de plotar os gráficos com a PlotlyJS é preciso definir suas configurações assim é preciso criar um objeto para o layout do gráfico e arrays com as linhas de cada gráfico 

                
let layout = {
  height: 250,
  autosize: true,
  margin: { b: 50, t: 30}
};

Criaremos uma função do tipo factory(fábrica) para criar um objeto no formato Trace (linha) que a biblioteca do PlotlyJS aceita 

                
// cria uma nova linha
function createTrace(name, valueTrace, color) {
  return {
    name,
    y: [valueTrace],
    type: 'line',
    mode: 'lines',
    line: { color },
  };
}

Depois de criada a função factory, podemos de colocar a linha gerada dentro de um array de linhas que será mandado como parametro para criar o gráfico 

                
let traces = [createTrace('sinal análogico', value, '#D00')];

Agora criaremos uma função para atualizar o gráfico que será o de voltagem Mais a frente passaremos essa função como callback para um setInterval 

                

function updateGraph() {
  Plotly.extendTraces('chart', { y: [[value]] }, [0]);
  x++;
  graphRelayout('chart', 'sinal analógico', 0, 1023);
  
  const valueLabel =  document.getElementById('value');
  valueLabel.innerText = 'Sinal analógico: ' + value.toFixed(2);
}

Além disso também é necessário redesenhar o gráfico para fazer com que ele sempre esteja se movimentando para frente, do contrário o X de nosso gráfico passaria do limite e não seria mais visível, assim definimos que ele sempre mostrára em uma escala do X - 50 até o X + 10, além disso definimos uma escala para o eixo Y. Definimos de 0 a 1023 que é a faixa de leitura analógica do Arduino 

                

// faz o redesenho do gráfico
function graphRelayout(divName, graphName, rangeMin, rangeMax) {
  Plotly.relayout(divName, {
    xaxis: {
      showticklabels: false,
      range: [x - 50, x + 10],
    },
    yaxis: {
      title: graphName,
      range: [rangeMin, rangeMax],
    },
  });
}

Agora definiremos uma função a ser executada assim que a página for carregada. 

                
window.onload = main;

function main() {
  socket.on('value', function (receivedData) {
    value = receivedData
  });
  Plotly.plot('chart', traces, layout); // plotar gráfico
  executingGraph = setInterval(updateGraph, 100); // atualiza o gráfico a cada 100 milisegundos
}

Definiremos uma função para pausar ou retomar o gráfico, funcionalidade importante para caso o usuário deseje visualizar os valores ou mesmo fazer uma captura de tela 

                
// função para pausar ou retomar o gráfico
function pauseResume() {
  pause = !pause;
  if (pause) {
    document.getElementById('pause-resume').innerText = 'Retomar';
    clearInterval(executingGraph); // pausa primeiro gráfico
  } else {
    document.getElementById('pause-resume').innerText = 'Pausar';
    executingGraph = setInterval(updateGraph, 100); // retoma primeiro gráfico
  }
}

Finalizando o servidor

Depois de criar todo o front-end da aplicação, está na hora de terminarmos o servidor e escrevermos todo o código referente ao Arduino e ao Socket.io do lado do servidor. Nesse tutorial é criado uma função IIFE denominada main para ser a porta de entrada do servidor Após isso usaremos a classe Board do Arduino para instanciar um objeto referente à placa do Arduino. Para o contrutor da classe passamos o caminho da porta USB que o Arduino está conectado Podemos verificar qual porta colocar, abrindo a IDE do Arduino e sequindo as opções
No Windows teremos como caminho para porta algo como: COM1, COM2 ou COM3, para o sistemas baseados em Linux algo como /dev/ttyUSB0.
Como esse tutorial foi reproduzido no Linux Ubuntu, foi colocado a /dev/ttyUSB0. 

                
(function main() {
  const arduino = new jFive.Board({ port: "/dev/ttyUSB0" });
})();

Não podemos fazer as leituras analógicas do Arduino imediatamente, uma vez que quando o script é executado no NodeJS, é levado um tempo até que a placa seja devidamente reconhecida. Para isso somente criaremos o objeto referente à leitura analógica somente quando o Arduino estiver pronto. Assim colocamos o código dentro do listener arduino.on('ready', callback) Usaremos a classe Sensor do johnny-five que pode servir para definir objetos para leitura analógica do Arduino. Passamos como parametros a porta analógica do Arduino A0 e a frequência de leitura analógica 

                
arduino.on('ready', function() {
  console.log('Arduino pronto');
  const analogSensor = new jFive.Sensor('A0', { freq: 250}); 
});

Uma vez instanciado o objeto referente ao sensor colocado no arduino já podemos começar a ler seus valores análgicos e mandar para a parte frontend da aplicação. Para troca de mensagens em tempo real entre servidor e cliente, começamos trabalhar com websockets através do Socket.io.

a função on() do socket.io permite definirmos um bloco de código a ser executado quando algum evento acontecer desta forma, quando o evento connection acontecer significa que o cliente conectou ao nosso servidor, a partir desse momento que os dados capturados do sensor do Arduino poderão ser enviados para o cliente. Desta forma edite o server.js.

Para um envio continuo dos dados para o cliente usaremos a função setInterval do JavaScript que permite que uma função seja executada a cada determinado período de tempo. 

                
// quando o cliente conectar
io.on('connection', function(socket) { 
  console.log('Mandando dados para ' + socket.id);
  setInterval(function() {
    socket.emit('value', analogSensor.value)
  }, 100);
});

Por fim precisamos definir a porta que nossa aplicação será servida Neste exemplo usaremos a porta 8080 que foi anteriormente definida pela constante port 

                
// ouvir na porta declarada 
http.listen(port, function() {
  console.log('============ SISTEMA PRONTO ============');
  console.log(`   Abrir em: http://localhost:${port}`);
  console.log('>> ========================================');
});

Uma vez terminada, o estado final da nossa aplicação do lado do servidor será da seguinte forma: 

                
const express = require("express"); // importando o Express 
const app = express();
const http = require("http").createServer(app);
const io = require("socket.io")(http); // importando Socket.io
const jFive = require("johnny-five"); // importando o Johnny-five
const path = require('path'); // será utilizado para fazer o express reconhecer o caminho 
const port = 8080;

app.use(express.static(path.resolve(__dirname + "/../frontend"))); // atender requisições com pasta a frontend

(function main() {
  const arduino = new jFive.Board({ port: "/dev/ttyUSB0" });
  arduino.on('ready', function() {
    console.log('Arduino pronto');
    const analogSensor = new jFive.Sensor('A0', { freq: 250}); 
    io.on('connection', function(socket) { 
      console.log('Mandando dados para ' + socket.id);
      setInterval(function() {
        socket.emit('value', analogSensor.value)
      }, 100);
    });
    // ouvir na porta declarada 
    http.listen(port, function() {
      console.log('============ SISTEMA PRONTO ============');
      console.log(`   Abrir em: http://localhost:${port}`);
      console.log('>> ========================================');
    });
  });
})();
